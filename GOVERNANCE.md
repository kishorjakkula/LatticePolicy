# Governance

LatticePolicy is currently maintained by the project maintainers.

## Decision Making

Maintainers guide roadmap, architecture, releases, and contribution review. Technical decisions should be documented in issues, pull requests, or `docs/` when they affect public APIs, extension points, deployment, or data model behavior.

## Maintainer Responsibilities

- Review pull requests and issues.
- Protect framework-level extension points from carrier-specific coupling.
- Keep documentation accurate for supported workflows.
- Coordinate security vulnerability handling.
- Define release scope and compatibility expectations.

## Compatibility

The project is pre-1.0. Breaking changes may occur, but they should be documented clearly with migration notes when they affect public APIs, schemas, tenant configuration, or data model contracts.
